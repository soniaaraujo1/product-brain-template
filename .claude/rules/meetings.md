# Working in Meetings/

- Name notes `YYYY-MM-DD-short-topic.md`. Use `_TEMPLATE.md`.
- Don't just summarise — **route** extracted items: decisions → `Decisions/Decision-Log.md`;
  actions → `To-Do.md`; open questions/risks → `Risks-and-Assumptions.md`.
- Never invent attendees, dates, or decisions; mark reconstructed/uncertain items `(confirm)`.
- `/capture-meeting` does this end-to-end.
